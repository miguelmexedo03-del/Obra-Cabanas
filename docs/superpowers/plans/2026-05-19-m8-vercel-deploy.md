# Deploy M8 — Vercel Production Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy `obra-cabanas-app/` to Vercel production with automatic CI/CD on push to `main`.

**Architecture:** Vercel CLI links the Next.js 16 app (inside subdirectory `obra-cabanas-app/`) to a Vercel project with Root Directory set to `obra-cabanas-app`. GitHub Integration handles deploys automatically — push to `main` triggers production, any other branch gets a preview URL. Env vars are stored in Vercel (never committed).

**Tech Stack:** Vercel CLI (v39+), Next.js 16 (App Router), Supabase Cloud (Postgres + Auth)

- [ ] **Step 3: Set Root Directory in Vercel dashboard**

> **Why this matters:** `obra-cabanas-app/.git` tracks this subdirectory, but the GitHub remote's root is the `obra-cabanas-app/` folder itself. Setting Root Directory tells Vercel to build from this folder. Without it, Vercel might look one level up and find no `package.json`.

### Task 4: Configure environment variables

**Files:**
- None (stored in Vercel — never in the repo or `.env.local`)

Run all commands from inside `obra-cabanas-app/`. Each `vercel env add` call prompts for the value (paste and press Enter) then asks which environments to apply to (use spacebar to select, Enter to confirm).

Select **both `Production` and `Preview`** for all vars unless noted.

- [ ] **Step 1: Add NEXT_PUBLIC_SUPABASE_URL**

```bash
vercel env add NEXT_PUBLIC_SUPABASE_URL
```
Value: your Supabase project URL, e.g. `https://abcdefghij.supabase.co`
Environments: Production + Preview

- [ ] **Step 2: Add NEXT_PUBLIC_SUPABASE_ANON_KEY**

```bash
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY
```
Value: your anon key (starts with `eyJ...`) — find it in Supabase dashboard → Project Settings → API → `anon public`
Environments: Production + Preview

- [ ] **Step 3: Add SUPABASE_SERVICE_ROLE_KEY**

```bash
vercel env add SUPABASE_SERVICE_ROLE_KEY
```
Value: your service role key (starts with `eyJ...`) — find it in Supabase → Project Settings → API → `service_role`
Environments: Production + Preview

> **Critical:** This key bypasses Row Level Security. It is only used in Server Actions (server-side). Never prefix it with `NEXT_PUBLIC_` — doing so would leak it to every browser that loads the app.

- [ ] **Step 4: Add NEXT_PUBLIC_APP_NAME**

```bash
vercel env add NEXT_PUBLIC_APP_NAME
```
Value: `Obra Cabanas`
Environments: Production + Preview

- [ ] **Step 5: Add NEXT_TELEMETRY_DISABLED**

```bash
vercel env add NEXT_TELEMETRY_DISABLED
```
Value: `1`
Environments: Production + Preview

- [ ] **Step 6: Verify all vars are set**

```bash
vercel env ls
```
Expected: table showing 5 variables, each with Production and Preview environments listed.

> `NEXT_PUBLIC_APP_URL` is added after the first deploy in Task 6, once the actual URL is known.

### Task 6: Supabase auth redirect URLs + NEXT_PUBLIC_APP_URL

**Files:**
- None (Supabase dashboard configuration + one Vercel env var)

- [ ] **Step 1: Note your exact production URL**

From Task 5 Step 3 output, the URL looks like one of:
- `https://obra-cabanas-app.vercel.app`
- `https://obra-cabanas-app-<hash>.vercel.app`

Use this exact URL in the steps below.

- [ ] **Step 2: Configure Supabase auth redirect URLs**

Open https://supabase.com/dashboard → your **production** project → **Authentication** → **URL Configuration**.

Set:
- **Site URL:** `https://<your-vercel-url>` (exact URL from Step 1)
- **Redirect URLs** — add both of these (one per line, click + to add each):
  - `https://<your-vercel-url>/**`
  - `https://obra-cabanas-app-git-*.vercel.app/**`

Click **Save**.

> The `**` wildcard covers all paths. The `git-*` pattern covers every preview deployment Vercel auto-generates per branch — without it, auth redirects fail on preview URLs.

- [ ] **Step 3: Add NEXT_PUBLIC_APP_URL**

```bash
vercel env add NEXT_PUBLIC_APP_URL
```
Value: `https://<your-vercel-url>` (e.g., `https://obra-cabanas-app.vercel.app`)
Environments: Production + Preview

- [ ] **Step 4: Redeploy to pick up new env var**

```bash
vercel deploy --prod
```
Expected: new production deploy completes successfully.
